# Übung 9 – Abstrakte Klassen und abstrakte Methoden

Bisher besitzt ihr die Klassen:

* `Computer`
* `Laptop`
* `DesktopComputer`

Dabei ist `Computer` die Oberklasse.

Momentan kann jedoch auch folgendes Objekt erzeugt werden:

```java
Computer computer = new Computer(...);
```

> Macht ein allgemeiner **Computer** in eurem Programm überhaupt Sinn?

Oder gibt es immer nur konkrete Geräte wie

* Laptop
* DesktopComputer

## Aufgabe 1 – Computer abstrakt machen

Ändere die Klasse `Computer` so, dass sie **nicht mehr direkt instanziiert** werden kann.

1. Mache die Klasse `Computer` zu einer abstrakten Klasse.
2. Versuche anschließend folgenden Code zu kompilieren:

```java
Computer computer = new Computer(...);
```

3. Überprüfe anschließend, ob folgende Objekte weiterhin erzeugt werden können:

```java
Laptop laptop = new Laptop(...);

DesktopComputer desktop = new DesktopComputer(...);
```

Die Methode

```java
gibInformationenAus()
```

existiert momentan bereits in den Unterklassen.

* Kann die Oberklasse `Computer` überhaupt wissen, wie ein `Laptop` seine Informationen ausgeben soll?
* Kann sie wissen, welche zusätzlichen Informationen ein `DesktopComputer` besitzt?

## Aufgabe 2 – Abstrakte Methode

Die Oberklasse soll lediglich festlegen, **dass** jedes Computerobjekt Informationen ausgeben können muss.

Wie die Ausgabe genau aussieht, entscheidet jedoch jede Unterklasse selbst.

1. Deklariere in der Klasse `Computer` eine abstrakte Methode:

```java
public abstract void gibInformationenAus();
```

2. Entferne gegebenenfalls die bisherige Implementierung.

3. Beobachte die Compilerfehler.

Warum entstehen diese?

## Aufgabe 3 – Methoden implementieren

Implementiere nun die abstrakte Methode in beiden Unterklassen.

### Laptop

Die Ausgabe soll zusätzlich enthalten:

* Akkustand
* Bildschirmgröße

### DesktopComputer

Die Ausgabe soll zusätzlich enthalten:

* RGB-Beleuchtung
* Gehäusetyp

Verwende dabei die Annotation

```java
@Override
```

## Aufgabe 4 – Polymorphie überprüfen

Lege erneut ein Array vom Typ

```java
Computer[]
```

an.

Speichere darin mehrere Objekte der Klassen

* `Laptop`
* `DesktopComputer`

Gib anschließend alle Geräte mit einer Schleife aus.

```java
for (Computer computer : computerListe) {
    computer.gibInformationenAus();
}
```

Beobachte:

* Welche Methode wird jeweils aufgerufen?
* Warum funktioniert das, obwohl das Array vom Typ `Computer[]` ist?

# Reflexionsfragen

1. Was ist eine abstrakte Klasse?
2. Worin unterscheidet sich eine abstrakte Klasse von einer normalen Klasse?
3. Warum können von einer abstrakten Klasse keine Objekte erzeugt werden?
4. Was ist eine abstrakte Methode?
5. Warum besitzt eine abstrakte Methode keinen Methodenrumpf?
6. Warum müssen `Laptop` und `DesktopComputer` die Methode `gibInformationenAus()` implementieren?
7. Welche Vorteile bieten abstrakte Klassen gegenüber einer normalen Oberklasse?
