# Übung 10 – Interfaces

# Ausgangssituation

Bisher besitzt ihr die Klassen

* `Computer`
* `Laptop`
* `DesktopComputer`

Außerdem kann jeder Computer ein- und ausgeschaltet werden.

Nun soll die IT-Abteilung zusätzlich folgende Geräte verwalten:

* Monitor
* Drucker

Auch diese Geräte können ein- und ausgeschaltet werden.

Aber:

> Ist ein Monitor ein Computer?

> Ist ein Drucker ein Computer?

Nein.

Trotzdem besitzen alle Geräte dieselbe Fähigkeit:

> Sie können ein- und ausgeschaltet werden.

## Aufgabe 1 – Das Interface erstellen

Erstellt ein Interface

```java
Einschaltbar
```

Das Interface soll folgende Methoden enthalten:

```java
void einschalten();

void ausschalten();
```

## Aufgabe 2 – Interface implementieren

Die abstrakte Klasse

```java
Computer
```

soll das Interface

```java
Einschaltbar
```

implementieren.

Ergänze dazu die Klassendeklaration.

## Aufgabe 3 – Neue Klasse Monitor

Erstelle eine Klasse

```java
Monitor
```

mit folgenden Attributen:

* `hersteller`
* `groesse`
* `eingeschaltet`

Die Klasse soll ebenfalls das Interface

```java
Einschaltbar
```

implementieren.

Implementiere beide Methoden.

## Aufgabe 4 – Weitere Klasse Drucker

Erstelle eine Klasse

```java
Drucker
```

mit folgenden Attributen:

* `hersteller`
* `modell`
* `papierVohanden`

Auch diese Klasse soll das Interface

```java
Einschaltbar
```

implementieren.

## Aufgabe 5 – Gemeinsam nutzen

Erzeuge Objekte der Klassen:

* `Laptop`
* `DesktopComputer`
* `Monitor`
* `Drucker`

Speichere diese Objekte in einem Array vom Typ

```java
Einschaltbar[]
```

Überlege zunächst:

> Warum funktioniert das?

Anschließend:

* Schalte alle Geräte mit einer Schleife ein.
* Schalte anschließend alle Geräte wieder aus.

## Aufgabe 6 – Vererbung oder Interface?

Ordne die folgenden Beziehungen zu.

Schreibe jeweils dazu:

* **Vererbung**
* **Interface**

| Beziehung                    | Lösung |
| ---------------------------- | ------ |
| Laptop ___ Computer          |        |
| DesktopComputer ___ Computer |        |
| Monitor ___ Einschaltbar     |        |
| Drucker ___ Einschaltbar     |        |
| Laptop ___ Einschaltbar      |        |


# Reflexionsfragen

1. Was beschreibt eine Vererbung?
2. Was beschreibt ein Interface?
3. Warum erbt ein Monitor nicht von `Computer`?
4. Warum kann ein Monitor trotzdem `Einschaltbar` implementieren?
5. Welche Gemeinsamkeit besitzen Computer, Monitor und Drucker?
6. Warum kann ein Objekt gleichzeitig von einer Klasse erben und zusätzlich mehrere Interfaces implementieren?

# Zusatzaufgabe

Einige Geräte können sich zusätzlich mit einem Netzwerk verbinden.

Erstelle das Interface

```java
Netzwerkfaehig
```

mit den Methoden

```java
void verbinden();

void trennen();
```

Überlege anschließend, welche Klassen dieses Interface implementieren sollten.

# Bonusaufgabe

Ein Multifunktionsdrucker kann

* eingeschaltet werden,
* sich mit dem Netzwerk verbinden.

Welche Interfaces müsste ein Multifunktionsdrucker implementieren?

Skizziere die Klassendeklaration.

## Merksatz

**Vererbung beschreibt eine IST-EIN-Beziehung.**

Beispiele:

* Ein Laptop **ist ein** Computer.
* Ein DesktopComputer **ist ein** Computer.

**Interfaces beschreiben Fähigkeiten (KANN).**

Beispiele:

* Ein Computer **kann** eingeschaltet werden.
* Ein Monitor **kann** eingeschaltet werden.
* Ein Drucker **kann** eingeschaltet werden.
